将本地文件同步到云端

在本地项目文件夹内操作
初始化仓库：
git init

连接远程仓库：
git remote add gitee https://gitee.com/naonano/xxx.git

将文件发送到暂存区：
git add .

提交本地暂存区：
git commit -m "first commit"

上传到仓库：
git push -u origin "master"
如果没有下拉过仓库代码想要直接用本地覆盖云端请使用：
git push -u --force origin "master"

其他常用操作

更新仓库：
git push gitee master
git push github master

查看远程仓库信息：
git remote -v

查看本地文件夹文件同步状态：
git status

删除与远程仓库关联：
git remote rm origin(这个名字的地方，根据自己的名字修改)

问题及解决

1. 对于error: failed to push some refsto‘远程仓库地址’
git pull --rebase origin master
git push -u origin master

2. 更新远程仓库时报错：![rejected] master -> master (fetch first)
 这是由于本地与远程仓库版本不一致导致

 方法一：
 先备份本地修改的文件，使用以下命令拉取远程仓库
 git pull --rebase origin master
 之后再重新同步
 git pull origin master

 方法二：
 强制同步(注意！会覆盖云端代码)：
 git push -f origin master 或 git push --force origin master
